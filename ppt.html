Your First C Program: "Hello, World!"

Let's write our very first C program! It's a tradition to start with "Hello, World!"


#include <stdio.h> // This line helps us talk to the screen!

int main() {       // This is where our program starts running.
    printf("Hello, World!\n"); // This tells the computer to show "Hello, World!"
    return 0;      // This tells the computer we're done and everything went well.
}

Every line has a special job!

How Computers Understand C: The Translator

Computers don't understand C directly. They need a translator called a compiler.

  1. You write C code (like a story in English).
  2. The compiler translates your C code into "machine code" (like a robot's language).
  3. The computer then runs the machine code!

Think of the compiler as a super-smart translator!

Setting Up Your C Tools

To write and run C programs, you'll need a few tools:

  • Text Editor / IDE: Where you type your code (like Notepad, VS Code, Code::Blocks).
  • Compiler: The translator (like GCC).

We'll help you get these set up on your computer!

Running Your First Program: Let's Go!

After writing your "Hello, World!" code, here's how to run it:

  1. Save: Save your code as `hello.c` (the `.c` is important!).
  2. Compile: Open your command line (like a text-based computer control panel) and type: gcc hello.c -o hello
  3. Run: Type: ./hello

You should see "Hello, World!" appear on your screen!

C's Vocabulary: Keywords & Identifiers

Keywords: Special Words

  • Words C already understands, like `int` or `return`.
  • You can't use them for other things.

Identifiers: Your Own Names

  • Names you choose for your "boxes" or "instructions."
  • Like `myAge`, `calculateScore`.
  • Must start with a letter or underscore.
  • Are case-sensitive (`myVar` is different from `myvar`).

C's Smallest Pieces: Tokens

Imagine a sentence. It's made of words, punctuation, and numbers. In C, these small pieces are called Tokens.

  • Keywords: (`int`, `if`, `for`)
  • Identifiers: (`myVariable`, `addNumbers`)
  • Constants: (`10`, `3.14`, `'A'`)
  • Strings: (`"Hello"`)
  • Operators: (`+`, `-`, `=`, `==`)
  • Punctuation: (`;`, `{`, `}`)

The compiler breaks your code into these tiny tokens!

Storing Information: Variables

Think of a variable as a named box where you can store different kinds of information.

  • Each box has a name (like `age`, `score`).
  • Each box can hold a value (like `10`, `100`).
  • The value inside the box can change!

int age = 12; // A box named 'age' holding the number 12
age = 13;     // Now the 'age' box holds 13!

Declaring Variables: Making a New Box

Before you can use a box, you need to tell C what kind of stuff it will hold. This is called declaring a variable.

  • You tell C the type of data (like `int` for whole numbers).
  • You give it a name.
  • You can also put something in it right away (initialize).

int myScore; // Declared a box for a whole number, named 'myScore'
myScore = 500; // Put 500 into the 'myScore' box

float price = 19.99f; // Declared a box for a decimal, named 'price', and put 19.99 in it

Different Types of Boxes: Data Types

Just like you have different containers for different things (a lunchbox for sandwiches, a jar for cookies), C has different data types for different kinds of information.

  • `int` for whole numbers (like `10`, `500`, `-3`).
  • `char` for single letters or symbols (like `'A'`, `'z'`, `'!'`).
  • `float` and `double` for numbers with decimals (like `3.14`, `99.99`).

`int`: Whole Numbers

The `int` data type is used to store whole numbers (integers).

  • No decimal points allowed!
  • Can be positive, negative, or zero.
  • Great for counting things, ages, scores.

int numberOfApples = 5;
int myAge = 10;
int temperature = -3;

`char`: Single Letters

The `char` data type is used to store a single character (a letter, a number, or a symbol).

  • Always put single characters in single quotes (`' '`).
  • Can also store very small whole numbers.

char firstLetter = 'J';
char grade = 'A';
char symbol = '#';

`float` & `double`: Decimal Numbers

When you need to store numbers with decimal points, use `float` or `double`.

  • `float` is good for most decimal numbers.
  • `double` is for super-precise decimal numbers (it uses more memory but is more accurate).

float pi = 3.14f;
double exactValue = 123.456789;
float price = 9.99f; // Add 'f' for float literals!

Fixed Boxes: Constants

Sometimes you have values that should never change in your program. These are called constants.

  • Like the number of days in a week (always 7).
  • Makes your code clearer and prevents mistakes.

Two ways to make constants:

  • `const` keyword: Makes a variable read-only.
  • `#define` preprocessor: Replaces a name with a value before your code even compiles.

#define MAX_SCORE 1000 // MAX_SCORE is always 1000
const int DAYS_IN_WEEK = 7; // DAYS_IN_WEEK is always 7

Getting Input: `scanf()`

How do we let the user type something into our program? We use the `scanf()` function!

  • It waits for the user to type something and press Enter.
  • We tell it what kind of data to expect (like `%d` for `int`).
  • We use `&` (the "address-of" symbol) to tell `scanf` *where* to put the value.

#include <stdio.h>
int main() {
    int favoriteNumber;
    printf("What's your favorite number? ");
    scanf("%d", &favoriteNumber); // Read an integer into 'favoriteNumber'
    printf("Your favorite number is: %d\n", favoriteNumber);
    return 0;
}

Showing Output: `printf()`

We've already met `printf()`! It's how our program talks back to us by showing text on the screen.

  • You put the text you want to show inside `""`.
  • Use Format Specifiers like `%d`, `%f`, `%c`, `%s` to show values from variables.
  • Use Escape Sequences like `\n` for a new line, `\t` for a tab.

#include <stdio.h>
int main() {
    int age = 10;
    printf("Hello, bootcampers!\n");
    printf("You are %d years old.\n", age); // Using %d for 'age'
    return 0;
}

Doing Math: Arithmetic Operators

Computers are great at math! C has special symbols for basic math operations:

  • `+` (Addition)
  • `-` (Subtraction)
  • `*` (Multiplication)
  • `/` (Division)
  • `%` (Modulo - gives the remainder after division)

int num1 = 10;
int num2 = 3;
int sum = num1 + num2; // sum will be 13
int remainder = num1 % num2; // remainder will be 1 (10 divided by 3 is 3 with 1 left over)

Comparing Things: Relational Operators

Sometimes you need to compare values, like "Is this number bigger than that one?" or "Are these two numbers the same?"

  • `==` (Is equal to?)
  • `!=` (Is NOT equal to?)
  • `<` (Is less than?)
  • `>` (Is greater than?)
  • `<=` (Is less than or equal to?)
  • `>=` (Is greater than or equal to?)

int score = 90;
if (score >= 90) {
    printf("Great job!\n");
}

Combining Conditions: Logical Operators

What if you have multiple conditions? Like "If it's sunny AND it's warm..."

  • `&&` (AND): Both conditions must be true.
  • `||` (OR): At least one condition must be true.
  • `!` (NOT): Reverses a condition (true becomes false, false becomes true).

int age = 10;
char hasTicket = 'Y';
if (age >= 8 && hasTicket == 'Y') {
    printf("You can enter the show!\n");
}

Smart Programs: Decision Making

Imagine your program is at a crossroads. It needs to decide which path to take based on certain rules.

  • Decision making lets your program choose different actions.
  • It's like telling your program: "If this is true, do A; otherwise, do B."

This makes your programs much more intelligent and useful!

The `if` Statement: "If this, then that!"

The simplest decision-maker. It checks a condition, and if it's true, it does something.


int sunny = 1; // 1 means true, 0 means false
if (sunny == 1) {
    printf("Let's go play outside!\n");
}
  • If `sunny` is true, the message prints.
  • If `sunny` is false, nothing happens.

The `if-else` Statement: "If this, or else that!"

This gives your program two choices: one if the condition is true, and another if it's false.


int isRaining = 0;
if (isRaining == 1) {
    printf("Grab an umbrella!\n");
} else {
    printf("No umbrella needed!\n");
}
  • One path will *always* be taken.

Multiple Choices: `if-else if-else` Chain

What if you have many options? Like grading a test: A, B, C, or F?


int score = 85;
if (score >= 90) {
    printf("Grade: A\n");
} else if (score >= 80) {
    printf("Grade: B\n");
} else if (score >= 70) {
    printf("Grade: C\n");
} else {
    printf("Grade: F\n");
}
  • The program checks conditions one by one.
  • It stops at the first one that is true.

Nested Decisions: Decisions Inside Decisions!

You can put an `if` statement inside another `if` statement. This is called nesting.


int hasKey = 1;
int doorLocked = 1;
if (hasKey == 1) {
    if (doorLocked == 1) {
        printf("Unlocking the door!\n");
    } else {
        printf("Door is already open.\n");
    }
} else {
    printf("No key, can't open.\n");
}

It helps make very specific choices!

Choosing from Many: The `switch` Statement

When you have a single variable and many possible actions based on its value, `switch` is super handy!


char dayCode = 'M';
switch (dayCode) {
    case 'M':
        printf("It's Monday!\n");
        break; // Don't forget break!
    case 'T':
        printf("It's Tuesday!\n");
        break;
    default:
        printf("Unknown day.\n");
}

`switch` with `case` and `break`

  • `switch (expression)`: The value we are checking.
  • `case value:`: If the expression matches this value, do the code here.
  • `break;`: This is super important! It tells the `switch` to stop and jump to the end.
  • Without `break;`, the program will keep running into the next `case`! (This is called "fall-through" and can cause bugs!)

`switch` Default Case: The Catch-All

What if the `switch` expression doesn't match any of your `case` values?

  • The `default:` case is like the "else" for `switch`.
  • It runs if no other `case` matches.
  • It's optional, but good for handling unexpected values.

int choice = 5;
switch (choice) {
    case 1: printf("You chose option 1.\n"); break;
    case 2: printf("You chose option 2.\n"); break;
    default: printf("Please choose 1 or 2.\n");
}

Doing Things Many Times: Loops

Imagine you need to print "Hello!" 100 times. Would you type `printf("Hello!\n");` 100 times?

No way! Loops let your program repeat actions over and over again. This saves a lot of typing!

  • Great for counting.
  • Great for doing the same thing for many items.

The `while` Loop: "Keep going while..."

A `while` loop keeps repeating as long as a condition is true.

  • It checks the condition before each time it runs.
  • If the condition starts as false, it won't run even once!

int count = 0;
while (count < 5) { // While count is less than 5...
    printf("Count: %d\n", count);
    count = count + 1; // Don't forget to change count, or it loops forever!
}
// Output: Count: 0, Count: 1, Count: 2, Count: 3, Count: 4

`while` Loop Example: Counting Up

Let's use a `while` loop to count from 1 to 5:


#include <stdio.h>
int main() {
    int i = 1;
    while (i <= 5) {
        printf("%d ", i);
        i++; // Same as i = i + 1;
    }
    printf("\n");
    return 0;
}
// Output: 1 2 3 4 5

Remember to change the condition inside the loop, or it will never stop!

The `do-while` Loop: "Do once, then keep going while..."

A `do-while` loop is a bit different:

  • It always runs at least once.
  • It checks the condition after each time it runs.

int num = 0;
do {
    printf("Enter 5 to exit: ");
    scanf("%d", &num);
} while (num != 5); // Keep doing this until num is 5
// This loop will always ask for input at least once.

`do-while` Loop Example: Simple Game Menu

Imagine a simple game menu that always shows up at least once:


#include <stdio.h>
int main() {
    int choice;
    do {
        printf("\n--- Game Menu ---\n");
        printf("1. Play Game\n");
        printf("2. Options\n");
        printf("3. Exit\n");
        printf("Enter your choice: ");
        scanf("%d", &choice);
    } while (choice != 3);
    printf("Thanks for playing!\n");
    return 0;
}

The `for` Loop: "For a specific number of times..."

The `for` loop is perfect when you know exactly how many times you want to repeat something.


for (int i = 0; i < 5; i++) {
    // 1. Start: int i = 0; (Do this once at the beginning)
    // 2. Keep going while: i < 5; (Check this before each run)
    // 3. After each run: i++ (Do this after each run)
    printf("Loop %d\n", i);
}
// Output: Loop 0, Loop 1, Loop 2, Loop 3, Loop 4

It's very tidy!

`for` Loop Example: Times Table

Let's make a simple times table for the number 7!


#include <stdio.h>
int main() {
    int number = 7;
    printf("--- %d Times Table ---\n", number);
    for (int i = 1; i <= 10; i++) {
        printf("%d x %d = %d\n", number, i, number * i);
    }
    return 0;
}

Stopping Loops Early: `break`

Sometimes you need to stop a loop *before* it finishes all its planned runs. Use `break`!

  • `break` immediately jumps out of the loop.

for (int i = 0; i < 10; i++) {
    printf("%d ", i);
    if (i == 4) {
        printf("\nFound 4! Breaking loop.\n");
        break; // Loop stops here
    }
}
// Output: 0 1 2 3 4 (then the message and loop ends)

Skipping an Iteration: `continue`

What if you want to skip *just one* turn of the loop and go to the next one? Use `continue`!

  • `continue` skips the rest of the current loop's code.
  • Then it goes straight to the next turn (next iteration).

for (int i = 0; i < 5; i++) {
    if (i == 2) {
        printf(" (Skipping 2) ");
        continue; // Skips printing %d for i=2
    }
    printf("%d ", i);
}
// Output: 0 1 (Skipping 2) 3 4

Building Blocks: Functions

Imagine building with LEGOs. You don't build the whole castle at once, right?

You build small parts: a wall, a tower, a gate. In programming, these small, reusable parts are called Functions.

  • A function is a block of code that does one specific job.
  • You can use it over and over again.

Why Use Functions?

  • Stay Organized: Keeps your code neat and tidy.
  • Reuse Code: Write code once, use it many times!
  • Easier to Understand: Each function does a clear job.
  • Easier to Fix: If something breaks, you know which "block" to check.

Functions make your programming life much easier!

Function Declaration: The Plan

Before you use a function, you often need to tell C about it. This is like making a plan or a "prototype."

  • It tells C: "Hey, there's a function named `addNumbers` coming, it gives back an `int`, and it needs two `int`s."
  • It's usually put at the top of your code.

int addNumbers(int a, int b); // This is a function declaration!

Function Definition: The Actual Work

This is where you write the actual code that the function will perform.

  • It includes the `return_type`, `function_name`, `parameters`, and the code block `{}`.

int addNumbers(int num1, int num2) { // This is the function definition!
    int sum = num1 + num2;
    return sum; // Send the 'sum' back to where the function was called
}

Calling Functions: Using Your Blocks

Once you've declared and defined a function, you can "call" it (use it) from other parts of your program, like from `main()`.


#include <stdio.h>

int addNumbers(int a, int b); // Declaration

int main() {
    int result = addNumbers(5, 3); // Calling the function!
    printf("The sum is: %d\n", result);
    return 0;
}

int addNumbers(int num1, int num2) { // Definition
    return num1 + num2;
}

Returning Values from Functions

Many functions do some work and then give you back a result. This is called "returning a value."

  • The `return` keyword sends a value back.
  • The `return_type` in the function's declaration tells you what kind of value it will send back.

int multiply(int x, int y) {
    int product = x * y;
    return product; // Returns the calculated product
}

int main() {
    int myResult = multiply(4, 6); // myResult will be 24
    printf("Result: %d\n", myResult);
    return 0;
}

Functions with No Return (`void`)

Some functions just do a job, like printing something, and don't need to give back a value. For these, we use `void` as the return type.


void sayHello() { // 'void' means no value is returned
    printf("Hello from the function!\n");
}

int main() {
    sayHello(); // Just calls the function to do its job
    return 0;
}

They just perform an action!

Passing Information: Pass by Value

When you send a variable to a function, C usually makes a copy of that variable's value. This is called "pass by value."

  • The function gets its own copy to play with.
  • If the function changes its copy, the original variable outside the function is not changed.

void changeNumber(int x) { // 'x' is a copy of the original number
    x = x + 10;
    printf("Inside function, x is: %d\n", x);
}

int main() {
    int myNum = 5;
    printf("Before function, myNum is: %d\n", myNum); // 5
    changeNumber(myNum); // Pass a copy of myNum
    printf("After function, myNum is: %d\n", myNum); // Still 5!
    return 0;
}

Recursion: Functions Calling Themselves

Imagine a set of Russian nesting dolls, each doll holding a smaller version of itself. Recursion is like that!

  • A function calls itself to solve a problem.
  • It needs a "stop" rule (base case) so it doesn't call itself forever!
  • Great for problems that can be broken into smaller, similar problems.

int countdown(int n) {
    if (n == 0) { // Base case: stop when n is 0
        printf("Blast off!\n");
        return 0;
    }
    printf("%d...\n", n);
    return countdown(n - 1); // Recursive step: call itself with a smaller number
}

int main() {
    countdown(3); // Output: 3... 2... 1... Blast off!
    return 0;
}

Storing Many Things: Arrays

Imagine you have a list of similar things, like scores in a game, or names of your friends.

An Array is like a row of connected boxes, all holding the *same type* of stuff.

  • All elements must be the same data type (e.g., all `int`s, or all `char`s).
  • They are stored right next to each other in the computer's memory.

One-Dimensional Arrays: Lists of Data

This is the simplest kind of array, like a single list of items.


int scores[5]; // An array named 'scores' that can hold 5 integers
  • `int` is the type of data.
  • `scores` is the name of our array.
  • `[5]` means it has 5 spots (or "elements").

Accessing Array Elements: Finding Your Box

How do you get to a specific box in the row?

  • You use its index (its position number).
  • In C, the first box is always at index 0, not 1!
  • So, for an array of 5 elements, the indices are 0, 1, 2, 3, 4.

int numbers[3] = {10, 20, 30};
printf("First number: %d\n", numbers[0]); // Output: 10
printf("Third number: %d\n", numbers[2]); // Output: 30

numbers[1] = 25; // Change the second number to 25

Initializing Arrays: Filling Your Boxes

You can put values into your array boxes when you declare them:


int scores[4] = {95, 80, 75, 90}; // All 4 elements filled

int levels[] = {1, 2, 3}; // C figures out the size (3 elements)

int emptyArray[5] = {0}; // Initializes all 5 elements to 0

If you don't fill all of them, the rest get filled with 0s (for numbers).

Multi-Dimensional Arrays: Grids of Data

Imagine a spreadsheet or a game board (like Tic-Tac-Toe). That's a 2D array!

  • It's like an array of arrays.
  • You need two indices: one for the row, one for the column.

int gameBoard[3][3]; // A 3x3 game board
gameBoard[0][0] = 1; // Top-left corner
gameBoard[1][2] = 5; // Middle row, right column

Strings: Words and Sentences in C

In C, there's no special "string" data type like in some other languages.

A string is just an array of characters!


char greeting[] = "Hello"; // An array of characters: 'H', 'e', 'l', 'l', 'o'

The computer knows it's a string because of a special hidden character at the end...

The Null Terminator: The String's End Mark! (`\0`)

Every string in C *must* end with a special character called the null terminator. It looks like `\0`.

  • It tells C functions where the string finishes.
  • When you write `"Hello"`, C actually stores `'H', 'e', 'l', 'l', 'o', '\0'`.
  • So, `"Hello"` (5 letters) needs 6 spaces in memory!

char name[6] = {'B', 'o', 'b', '\0'}; // Correct way to manually terminate
printf("%s\n", name); // Prints "Bob"

String Input/Output

How do we get strings from the user or print them?

  • `printf("%s", myString);`: Prints a string.
  • `fgets(myString, size, stdin);`: Reads a line of text from the keyboard (safer than `gets()`).
  • `puts(myString);`: Prints a string and adds a new line automatically.

#include <stdio.h>
int main() {
    char city[20];
    printf("What city are you from? ");
    fgets(city, sizeof(city), stdin); // Reads input safely
    printf("You are from: %s", city); // %s prints the string
    return 0;
}

String Operations: Length (`strlen`)

How long is a string? The `strlen()` function tells you!

  • It counts characters until it finds the `\0` (null terminator).
  • You need to `#include ` to use it.

#include <stdio.h>
#include <string.h> // For strlen

int main() {
    char word[] = "Coding";
    int len = strlen(word);
    printf("The word '%s' has %d letters.\n", word, len); // Output: 6
    return 0;
}

String Operations: Copying (`strcpy`, `strncpy`)

You can't just use `=` to copy strings! You need special functions:

  • `strcpy(destination, source)`: Copies the `source` string to `destination`.
  • `strncpy(destination, source, max_chars)`: Copies up to `max_chars`. This is safer because it helps prevent copying too much and overflowing your box!

#include <string.h>
int main() {
    char original[] = "Hello";
    char copy[10]; // Make sure 'copy' is big enough!
    strcpy(copy, original); // copy now holds "Hello"
    printf("Copy: %s\n", copy);
    return 0;
}

Computer's Memory: The "Address Book"

Imagine your computer's memory as a giant building with many, many apartments.

  • Each apartment has a unique address (like `Apartment 101`, `Apartment 205`).
  • Variables are like the people living in those apartments.
  • When you declare a variable, C finds an empty apartment for it.

Pointers are how we keep track of these addresses!

Pointers: Variables that Store Addresses

A pointer is a special type of variable that doesn't store a normal value (like a number or a letter).

Instead, it stores the memory address of another variable.

  • Think of a pointer as a sticky note that says: "Go to Apartment 101 to find the number 5!"

Declaring Pointers: Making an "Address Note"

To declare a pointer, you use an asterisk (`*`) before its name:


int *myPointer; // This is a pointer that can store the address of an 'int' variable.
  • The `int` tells C what *type of data* is at the address the pointer points to.

The `&` (Address-of) Operator: "Where is it?"

The `&` symbol (ampersand) is used to find the memory address of a variable.


int score = 100;
int *scoreAddress; // Declare a pointer

scoreAddress = &score; // 'scoreAddress' now holds the memory address of 'score'

printf("Score is: %d\n", score);
printf("Address of score: %p\n", (void*)scoreAddress); // %p prints an address

The `*` (Dereference) Operator: "What's inside?"

The `*` symbol (when used *after* declaring a pointer) is called the dereference operator.

  • It means "go to the address stored in this pointer and get the value there."

int value = 25;
int *valuePtr = &value; // valuePtr holds the address of 'value'

printf("Value using variable: %d\n", value);
printf("Value using pointer: %d\n", *valuePtr); // Dereference to get 25

*valuePtr = 30; // Change the value at the address! 'value' is now 30!
printf("New value: %d\n", value);

Passing by Reference: "Giving the Address"

Remember "pass by value" where functions get a copy?

With pointers, you can "pass by reference" â€“ you give the function the *actual address* of the variable.

  • This means the function can change the original variable!

void doubleIt(int *numberAddress) {
    *numberAddress = (*numberAddress) * 2; // Change the original value
}

int main() {
    int myScore = 10;
    printf("Before: %d\n", myScore); // 10
    doubleIt(&myScore); // Pass the address of myScore
    printf("After: %d\n", myScore); // 20! It changed!
    return 0;
}

Dynamic Memory: Getting More Memory When You Need It

What if you don't know how many "boxes" you'll need when you write your program?

  • Dynamic memory allocation lets your program ask for more memory *while it's running*!
  • The `malloc()` function (`memory allocate`) asks the computer for a block of memory.
  • It gives you back a pointer to that new memory.

#include <stdlib.h> // For malloc
int *myNumbers;
int count = 5;
myNumbers = (int *) malloc(count * sizeof(int)); // Ask for space for 5 integers

if (myNumbers == NULL) { // Always check if it worked!
    printf("Oh no, couldn't get memory!\n");
}

Giving Memory Back: `free`

When you're done with memory you asked for using `malloc`, you *must* give it back to the computer!

  • The `free()` function returns the memory.
  • If you don't `free()` memory, your program will keep holding onto it, even if it doesn't need it. This is called a memory leak.
  • Memory leaks can make your computer slow or even crash!

int *myNumbers = (int *) malloc(5 * sizeof(int));
// ... use myNumbers ...
free(myNumbers); // Give the memory back!
myNumbers = NULL; // Important: Set pointer to NULL after freeing!

Common Pointer Mistakes: Watch Out!

  • Dangling Pointer: A pointer that points to memory that has been `free`d. If you try to use it, your program might crash! Always set to `NULL` after `free`.
  • Null Pointer Dereference: Trying to use a pointer that is `NULL`. This also crashes your program! Always check if a pointer is `NULL` before using it.
  • Wild Pointer: A pointer that was never given a valid address. It points to a random place, which can cause weird bugs! Always initialize pointers (e.g., to `NULL`).

Grouping Different Things: Structures (`struct`)

Imagine you want to describe your favorite pet. It has a name (text), an age (number), and maybe a favorite toy (text).

A Structure (`struct`) lets you group different types of information together under one name!


struct Pet {
    char name[20]; // Name of the pet
    int age;        // Age of the pet
    float weight;   // Weight of the pet
};

Accessing Structure Members: Getting Info Out

Once you have a `struct` variable, you use the dot operator (`.`) to get to its individual pieces of information.


#include <string.h> // For strcpy
struct Pet myDog;
strcpy(myDog.name, "Buddy"); // Set the name
myDog.age = 5;               // Set the age
myDog.weight = 15.5f;

printf("My dog's name is %s, and he is %d years old.\n", myDog.name, myDog.age);

If you have a pointer to a struct, you use the arrow operator (`->`)!

Unions: Sharing Memory (Advanced Concept)

Unions are like special boxes where different items can sit in the *same spot*, but only one at a time.

  • It saves memory because it only makes space for the *biggest* item.
  • If you put an `int` in, then a `float`, the `int` data gets overwritten!
  • This is a more advanced concept, used when memory is super tight.

union Data {
    int i;
    float f;
};
union Data myData;
myData.i = 10; // myData now holds 10 as an integer
myData.f = 20.5f; // myData now holds 20.5 as a float (the int 10 is gone!)

Enumerations (`enum`): Naming Numbers

Sometimes you have a list of choices, like days of the week or colors. `enum` lets you give names to numbers, making your code easier to read.


enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY };
// MONDAY is 0, TUESDAY is 1, and so on by default!

enum Day today = WEDNESDAY;
printf("Today is day number: %d\n", today); // Output: 2

It's like creating your own special list of named constants!

`typedef`: Giving New Names to Types

The `typedef` keyword lets you create a new, shorter, or more descriptive name for an existing data type.

  • Makes your code cleaner, especially with `struct`s and pointers.

typedef int WholeNumber; // Now 'WholeNumber' is just another name for 'int'
WholeNumber age = 10;

typedef struct {
    int x;
    int y;
} Point; // Now you can just say 'Point' instead of 'struct Point'
Point myLocation = {10, 20};

Files: Saving Your Work

When your program finishes, all its variables disappear! What if you want to save data for later?

  • You can save data to a file on your computer's hard drive.
  • Files let your program remember things even after it closes.
  • Think of it like writing notes in a notebook.

Opening and Closing Files

Before you can read from or write to a file, you need to "open" it. When you're done, you "close" it.

  • `fopen()`: Opens a file. You tell it the file name and how you want to use it (read, write, append).
  • `fclose()`: Closes a file. This is super important to save your changes and clean up!

#include <stdio.h>
int main() {
    FILE *myFile; // This is a special pointer for files
    myFile = fopen("my_notes.txt", "w"); // Open for writing ('w')
    if (myFile == NULL) { // Always check if opening worked!
        printf("Error opening file!\n");
        return 1;
    }
    fprintf(myFile, "Hello, file!\n"); // Write to the file
    fclose(myFile); // Close the file
    printf("Notes saved!\n");
    return 0;
}

Reading from Files

Once a file is open for reading, you can get information from it.

  • `fgetc(file_pointer)`: Reads one character at a time.
  • `fgets(buffer, size, file_pointer)`: Reads a whole line of text.
  • `fscanf(file_pointer, format, ...)`: Reads formatted data (like numbers or words).

#include <stdio.h>
int main() {
    FILE *myFile = fopen("my_notes.txt", "r"); // Open for reading ('r')
    if (myFile == NULL) return 1;
    char line[100];
    if (fgets(line, sizeof(line), myFile) != NULL) {
        printf("Read from file: %s", line);
    }
    fclose(myFile);
    return 0;
}

Writing to Files

To put information into a file, you use writing functions.

  • `fputc(character, file_pointer)`: Writes one character.
  • `fputs(string, file_pointer)`: Writes a whole string.
  • `fprintf(file_pointer, format, ...)`: Writes formatted data (like `printf` but to a file).

#include <stdio.h>
int main() {
    FILE *myFile = fopen("scores.txt", "a"); // Open for appending ('a')
    if (myFile == NULL) return 1;
    int score = 95;
    fprintf(myFile, "Player Score: %d\n", score);
    fclose(myFile);
    printf("Score saved!\n");
    return 0;
}

Preprocessor Directives: Instructions Before Compiling

These are special commands that start with `#` (like `#include`). They tell the compiler to do something *before* it even starts translating your code.

  • `#include`: Brings in code from other files (like `stdio.h`).
  • `#define`: Lets you create shortcuts for numbers or small pieces of code.

#define MAX_HEALTH 100 // MAX_HEALTH will be replaced with 100 everywhere it's used.
int playerHealth = MAX_HEALTH;

Conditional Compilation: Smart Code Building

Sometimes you want parts of your code to only be included if certain conditions are true. This is called conditional compilation.

  • `#ifdef DEBUG`: If `DEBUG` is defined, include this code.
  • `#ifndef RELEASE`: If `RELEASE` is NOT defined, include this code.
  • `#endif`: Marks the end of the conditional block.

#define DEBUG_MODE
#ifdef DEBUG_MODE
    printf("Debugging is ON!\n");
#endif

Command Line Arguments: Talking to Your Program

You can give your program extra information right when you run it from the command line!


#include <stdio.h>
int main(int argc, char *argv[]) {
    // argc: how many words you typed (including program name)
    // argv: an array of those words (strings)
    printf("You typed %d words.\n", argc);
    printf("First word: %s\n", argv[0]); // This is usually the program's name
    if (argc > 1) {
        printf("Your first extra word: %s\n", argv[1]);
    }
    return 0;
}
// If you run: ./myprogram hello world
// Output: You typed 3 words. First word: ./myprogram. Your first extra word: hello

Function Pointers: Functions as Variables

This is a bit tricky, but super cool!

  • A function pointer is a variable that stores the *address* of a function.
  • It lets you choose which function to call *while your program is running*!
  • Imagine a remote control that can point to different TVs (functions) and turn them on.

void sayHi() { printf("Hi!\n"); }
void sayBye() { printf("Bye!\n"); }

int main() {
    void (*myFunctionPtr)(); // Declare a function pointer
    myFunctionPtr = sayHi; // Point it to sayHi
    myFunctionPtr(); // Call sayHi() using the pointer! (Output: Hi!)
    myFunctionPtr = sayBye; // Now point it to sayBye
    myFunctionPtr(); // Call sayBye()! (Output: Bye!)
    return 0;
}

Bitwise Operations: Tiny Controls

Computers store everything as 0s and 1s (bits). Bitwise operations let you control these individual bits!

  • Like tiny on/off switches.
  • Super useful for making programs very fast or when talking to hardware.
  • Operators: `&` (AND), `|` (OR), `^` (XOR), `~` (NOT), `<<` (Left Shift), `>>` (Right Shift).

unsigned char flags = 0b00000001; // Only the first light is ON
flags |= 0b00000010; // Turn on the second light (flags is now 0b00000011)
if (flags & 0b00000001) {
    printf("First light is ON!\n");
}

Debugging: Becoming a Bug Detective

What happens when your code doesn't work the way you want? That's a bug!

Debugging is the process of finding and fixing those bugs.

  • It's like being a detective, looking for clues in your code.
  • Don't worry, everyone gets bugs! It's part of learning.

Common Errors and How to Fix Them

  • Syntax Errors: Typos, missing semicolons, wrong curly braces.
    • Compiler will tell you! Read the error message carefully.
  • Logic Errors: Code runs, but does the wrong thing (e.g., calculator adds instead of subtracts).
    • Use `printf` to show values at different steps.
    • Step through code with a debugger.
  • Runtime Errors: Program crashes while running (e.g., trying to use a `NULL` pointer).
    • Check `if (pointer == NULL)` before using pointers.
    • Look for "Segmentation fault" messages.

Writing Clean Code: Making Your Code Easy to Read

Imagine reading a book with no paragraphs, no spaces, and strange words. Hard, right?

Clean code is like a well-written book for programmers!

  • Comments: Explain what your code does (`// like this`).
  • Good Names: Use clear names for variables (`playerScore` instead of `x`).
  • Neat Layout: Use spaces and tabs to make it easy to follow.
  • Break into Functions: Keep functions small and focused on one job.

Why Clean Code Matters

  • For You: You'll understand your own code later!
  • For Others: If you work in a team, others can understand and help.
  • Fewer Bugs: Clearer code means fewer mistakes.
  • Easier to Change: When you want to add new features, it's a breeze!

It's a superpower for teamwork and future you!
